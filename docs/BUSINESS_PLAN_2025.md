## 1. Executive Summary

### 1.1 서비스 개요

**UNIQN**은 홀덤 포커 산업에 특화된 **구인/구직 및 근무 관리 플랫폼**입니다.

포커펍, 홀덤클럽, 토너먼트 주최사(구인자)와 딜러, 플로어, 매니저 등 종사자(스태프)를 연결하고, 지원부터 근무 기록, 급여 정산까지 전 과정을 디지털화합니다.

| 항목 | 내용 |
|------|------|
| **서비스명** | UNIQN Mobile App |
| **버전** | v1.0.0 |
| **플랫폼** | iOS, Android, Web (단일 코드베이스) |
| **핵심 가치** | 신뢰성 있는 인력 매칭 + 자동화된 근무 관리 |

### 1.2 핵심 지표 요약

```
┌─────────────────────────────────────────────────────────────┐
│  타겟 시장       │  스태프 5,000~10,000명 / 구인자 500~1,000개  │
│  3년 누적 매출   │  8,220만원 (구인자 중심 수익 모델)           │
│  Year 1 목표    │  사용자 확보 우선 (무료 6개월 → 유료 전환)    │
│  Year 2 목표    │  월 평균 200만원 / 연간 2,400만원            │
│  순이익률       │  60% (Year 2 기준, 앱스토어 수수료 15% 차감) │
│  개발 진도      │  70% (인증 + 구인구직 + 관리자 완료)         │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 핵심 경쟁력

1. **시장 선점**: 홀덤 산업 전용 구인/구직 플랫폼 부재
2. **신뢰성**: PASS/카카오 본인인증 기반 실명 검증
3. **자동화**: QR 출퇴근 → 근무시간 자동 기록 → 정산 자동 계산
4. **확장성**: 단일 코드베이스로 3개 플랫폼 동시 지원

## 2. 시장 분석

### 2.1 홀덤 산업 현황

#### 시장 규모 및 성장성

| 지표 | 수치 | 비고 |
|------|------|------|
| **국내 홀덤펍 수** | 약 3,000개+ | 서울/경기 1,500개 추정 |
| **종사자 수** | 약 30,000명+ | 딜러, 플로어, 매니저 등 |
| **연평균 성장률** | 15~20% | 최근 5년 기준 |
| **시장 규모** | 약 5,000억원+ | 토너먼트 포함 |

#### 성장 동인

```
✅ MZ세대 유입 증가 - 전략 게임으로서의 포커 인식 확산
✅ 미디어 노출 증가 - OTT 드라마, 유튜브 콘텐츠
✅ 토너먼트 활성화 - 상금 규모 증가, 참가자 다양화
✅ 프로 딜러 수요 - 고급 서비스 니즈 증가
```

### 2.2 타겟 시장 세분화

#### Primary: 스태프 (Staff)

| 구분 | 내용 |
|------|------|
| **프로필** | 20~45세 포커룸 종사자, 일일/단기 근무 선호층 |
| **규모** | 서울/경기 5,000~10,000명 |
| **니즈** | 안정적 일자리 확보, 정확한 급여 정산, 스케줄 관리 |
| **비용** | **무료** (하트 리워드로 서비스 이용) |

#### Secondary: 구인자 (Employer)

| 구분 | 내용 |
|------|------|
| **프로필** | 포커펍/홀덤클럽 운영자, 토너먼트 주최사 |
| **규모** | 서울/경기 500~1,000개 업소 |
| **니즈** | 신뢰할 수 있는 인력 확보, 출결 관리, 정산 자동화 |
| **월 지출 의향** | 공고당 300~3,000원 (일반~긴급), 월 5~10건 공고 시 1.5~3만원 |

### 2.3 경쟁 환경 분석

#### 직접 경쟁사: 없음

현재 홀덤 산업에 특화된 구인/구직 플랫폼은 **부재**합니다.

#### 간접 경쟁사

| 서비스 | 강점 | 약점 | UNIQN 대비 |
|--------|------|------|-----------|
| **알바몬/알바천국** | 대중 인지도 | 홀덤 특화 기능 없음 | 산업 특화 |
| **카카오톡 오픈채팅** | 접근성 | 신원 검증 불가, 관리 불가 | 본인인증 |
| **업소별 자체 채용** | 직접 관리 | 비효율, 풀 제한적 | 플랫폼 효과 |

#### 경쟁 우위

```
1. 산업 특화 → 딜러/플로어/매니저 역할 구분, 토너먼트 공고 타입
2. 신뢰 시스템 → PASS/카카오 본인인증, 상호 신고 시스템
3. 자동화 → QR 출퇴근, 근무시간 자동 계산, 정산 자동화
4. 데이터 → 근무 이력, 평점, 매칭 최적화 기반
```

## 3. 비즈니스 모델

### 3.1 수익 모델 개요

UNIQN은 **하트/다이아 이중 포인트 시스템**을 통해 수익을 창출합니다.

```
┌─────────────────────────────────────────────────────────────┐
│                 하트/다이아 포인트 시스템                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   💖 하트 (Heart)                                           │
│   └─ 무료 획득 (출석, 이벤트, 리뷰, 초대)                    │
│   └─ 만료: 획득 후 90일                                     │
│                                                             │
│   💎 다이아 (Diamond)                                       │
│   └─ 유료 충전 (인앱 결제)                                   │
│   └─ 만료 없음                                              │
│                                                             │
│   가치: 1 포인트 = 300원 (하트/다이아 동일)                   │
│   사용 우선순위: 하트(만료 임박 순) → 다이아                   │
│                                                             │
│   핵심 수익원: 구인자의 공고 등록 (다이아 소비)                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```

│  영업 이익:        14,400,000원                             │
│  이익률:                  60.0%                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 8.6 BEP (손익분기점)

```
월 고정비: 40만원
평균 공고당 순수익: 약 850원 (앱스토어 수수료 15% 차감 후)

BEP = 400,000 / 850 ≈ 470건/월

→ 구인자 100명 × 월 5건 공고 시 손익분기 도달
→ M9 (완전 유료화 3개월차)에 BEP 도달 예상
```

### 9.1 리스크 매트릭스

| **기술 장애** | 중간 | 낮음 | 모니터링, 장애 대응 체계 |
| **보안 사고** | 높음 | 낮음 | 보안 감사, 암호화 강화 |

```
대응:

```

#### 경쟁사 진입

```
대응:
├── 빠른 사용자 확보로 네트워크 효과 구축
├── 지속적 기능 고도화
├── 충성 사용자 Lock-in (이력 데이터)
└── B2B 선점 (독점 계약)
```
